# Звіт з виконання проєкту: Прогнозування опадів за допомогою машинного навчання
Виконав: Іванисько Володимир 472 група

## 1. Мета та опис проєкту
Мета проєкту полягає у розробці програмного прототипу міні-сервісу для прогнозування наявності опадів на наступний день на основі історичних метеоданих. Проєкт реалізовано у вигляді інтерактивного веб-застосунку з використанням фреймворку Streamlit (Варіант А згідно з умовами завдання). Це дозволило повністю автоматизувати процес завантаження даних, навчання моделі та отримання результату в зручному користувацькому інтерфейсі.

## 2. Виконання поставлених завдань
У ході виконання роботи реалізовано всі вимоги завдання:
- Отримання даних: Налаштовано автоматичне завантаження метеоданих через HTTP-запити з відкритого сервісу Open-Meteo.
- Підготовка даних: Використано щоденні дані (формат daily), які програмно перетворюються у структуру pandas.DataFrame.
- Формування цільової змінної: Створено бінарну цільову змінну (класи 0 — немає опадів, 1 — є опади) шляхом аналізу показника precipitation_sum зі зсувом на один день вперед.
- Навчання моделі: Здійснено розбиття вибірки на навчальну та тестову (у пропорції 80/20) і застосовано алгоритм класифікації Random Forest.
- Оцінка якості: Розраховано базові метрики (Accuracy, Precision, Recall, F1-score) для оцінки точності класифікації; звіт перекладено українською мовою для зручності.
- Інтерфейс та прогноз: Розроблено застосунок, що дозволяє робити прогноз на завтра (видає клас та ймовірність у відсотках) за введеними вручну або автоматично завантаженими з інтернету поточними даними.

## 3. Джерело даних та їх підготовка
Для навчання моделі використано реальні історичні дані міста Київ (Широта: 50.45, Довгота: 30.52).
* Джерело: Відкритий архів API archive-api.open-meteo.com.
* Період: з 01.01.2025 по 11.03.2026 (понад 430 записів, що забезпечує достатній обсяг для якісного навчання).
* Ознаки (Features - X):
  - temperature_2m_max — Максимальна денна температура (°C)
  - temperature_2m_min — Мінімальна денна температура (°C)
  - wind_speed_10m_max — Максимальна швидкість вітру (км/год)
  - precipitation_sum — Сума опадів за поточний день (мм)

Останній рядок отриманого датасету автоматично видаляється (метод dropna), оскільки для нього об'єктивно неможливо сформувати цільову змінну на "завтра".

## 4. Модель машинного навчання
Як алгоритм класифікації обрано RandomForestClassifier (n_estimators=100, random_state=42) з бібліотеки scikit-learn. Цей вибір зумовлено його високою стійкістю до перенавчання та ефективністю у знаходженні прихованих нелінійних залежностей між погодними умовами. Після навчання об'єкт моделі зберігається у пам'яті сесії застосунку (st.session_state), щоб забезпечити швидкий доступ при прогнозуванні без необхідності повторного навчання.

## 5. Структура проєкту
Проєкт складається з наступних файлів:
* .streamlit/config.toml — налаштування колірної теми веб-інтерфейсу.
* app.py — головний скрипт (містить логіку інтерфейсу та машинне навчання).
* weather_daily.csv — локальний набір даних (створюється автоматично програмою).
* requirements.txt — перелік зовнішніх бібліотек Python.
* README.md — даний файл-звіт з інструкціями для перевірки.

## 6. Швидкий запуск та перевірка роботи (у VS Code)

Щоб зручно та швидко перевірити роботу, достатньо виконати 3 простих кроки прямо у вбудованому терміналі редактора (наприклад, VS Code):

Крок 1. Відкрийте проєкт
Завантажте проєкт та відкрийте папку `Weather_ML_Project` у редакторі коду. Відкрийте термінал (у VS Code: `Terminal -> New Terminal`).

Крок 2. Встановіть бібліотеки
Переконайтеся, що файл `requirements.txt` присутній у папці, і введіть команду для встановлення необхідних пакетів:
```bash
pip install -r requirements.txt

Крок 3. Запустіть застосунок
У тому ж терміналі введіть команду запуску:
streamlit run app.py
Після цього застосунок автоматично відкриється у браузері.
